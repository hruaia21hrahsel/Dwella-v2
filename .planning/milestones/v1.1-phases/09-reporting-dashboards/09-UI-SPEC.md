# Phase 9 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — React Native Paper + custom theme system |
| Preset | not applicable |
| Component library | React Native Paper v5 (`react-native-paper`) |
| Icon library | `@expo/vector-icons` — MaterialCommunityIcons |
| Font | System default (iOS: SF Pro, Android: Roboto) — no custom font loaded |

**Theme system:** `constants/theme.ts` (LightTheme / DarkTheme). All components must consume `useTheme()` from `lib/theme-context.tsx`. Never hardcode hex values — reference theme tokens.

**Charting library:** `victory-native` (recommended; `react-native-svg` v15.12.1 already installed as required peer dependency). Install via `npx expo install victory-native`. Provides VictoryBar, VictoryPie, VictoryLine, VictoryChart, VictoryArea — all SVG-based, no native module changes needed.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline separators |
| sm | 8px | Gap between chips, stat grid gap, card internal row gap |
| md | 16px | Screen horizontal padding, card internal padding |
| lg | 24px | Section gap (between P&L card and donut card) |
| xl | 32px | Empty state internal padding |
| 2xl | 48px | Scroll bottom padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Chart container height: P&L grouped bar chart = 200px tall; donut chart = 220px tall; occupancy chart = 180px tall; sparkline on property card = 40px tall
- KPI card grid: 2-column grid with 8px gap, each card padding 16px
- Touch targets for year/quarter/month selector chips: minimum 44px tall (accessibility requirement — D-06, D-07, D-08)
- Tooltip overlay: 8px padding inside tooltip bubble

Source: Derived from existing `dashboard/index.tsx` (padding: 16, gap: 8, marginTop: 24 section breaks) and `EmptyState.tsx` (padding: 32).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|

| Body | 16px | 400 | 1.5 (24px) |
| Heading | 16px | 700 | 1.2 (19px) |
| Subheading | 20px | 700 | 1.1 (22px) |
| Display | 22px | 700 | 1.1 (24px) |

**Only two weights are used across this entire phase: 400 (Regular) and 700 (Bold). No other weights (500, 600, 800) appear anywhere.**

**Specific assignments for this phase (all mapped to declared scale — no additional sizes):**
- Screen title (portfolio "Analytics" header): Display — 22px, weight 700, line-height 1.1
- Section heading ("P&L", "Expense Breakdown", "Tenant Reliability", "Occupancy"): Heading — 16px, weight 700, line-height 1.2
- KPI value on portfolio summary cards: Subheading — 20px, weight 700, color from status context (income=success, expense=error, net=conditional)
- KPI label below value: Label — 13px, weight 400, color `textSecondary`
- Chart axis tick labels: Label — 13px, weight 400, color `textSecondary`
- Tooltip value: Heading — 16px, weight 700, color `textPrimary`
- Tooltip label: Label — 13px, weight 400, color `textSecondary`
- Reliability table: tenant name 16px weight 700; percentage 16px weight 700; "avg days late" 13px weight 400
- Property summary card name: Heading — 16px, weight 700
- Property summary card sub-text (occupancy fraction, period): Label — 13px, weight 400, color `textSecondary`
- Empty state title: Heading — 16px, weight 700 (overrides existing EmptyState default of 17px — apply `titleStyle={{ fontSize: 16 }}`)
- Empty state subtitle: Body — 16px, weight 400, color `textDisabled`
- Selector chip text (year/granularity/quarter/month): Label — 13px, weight 400
- Year picker active year text: Heading — 16px, weight 700

**Note on existing EmptyState component:** The `EmptyState` component renders title at 17px by default. For this phase, pass `titleStyle={{ fontSize: 16 }}` to bring it within the declared scale. Do not modify the component globally.

Source: Extrapolated from `dashboard/index.tsx` styles (sectionTitle: 16px/700, selectorChipText: 13px) and remapped to 4-size / 2-weight scale per design contract. 14px, 11px, 12px, 15px, 17px do not appear in this phase. Weights 500, 600, 800 collapsed to 400 or 700 respectively.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `colors.background` (#F5F5F7 light / #1f1f1e dark) | Screen background, scroll container |
| Secondary (30%) | `colors.surface` (#FFFFFF light / #161616 dark) | All chart cards, KPI cards, property summary cards, reliability table rows |
| Accent (10%) | `colors.primary` (#009688 light / #4DB6AC dark) | Year picker active state, granularity chip active border, portfolio header gradient start, sparkline line color |
| Destructive | `colors.error` (#EF4444 light / #F87171 dark) | Expense bars in P&L chart, negative net P&L value, expense KPI value |

**Accent reserved for:** year picker active chip border + background tint, currently-selected granularity chip, portfolio header background gradient (`gradients.hero`), sparkline stroke on property cards. Not applied to buttons, table rows, or chart fills beyond these.

**Focal Points:**
- Portfolio screen: the 4-KPI grid row is the primary focal point — it is the first full-width element below the header and concentrates the highest-information-density data.
- Property report screen: the P&L bar chart is the primary focal point — it is the tallest chart (200px), appears first in the scroll order, and uses the highest-contrast color pair (income green / expense red).

**Chart-specific color assignments (locked):**
- P&L grouped bars: income bar = `colors.statusConfirmed` (#22C55E light); expense bar = `colors.error` (#EF4444 light). Both use theme tokens — dark mode uses the dark theme equivalents automatically.
- Donut chart segments: use `getCategoryColor()` from `lib/expenses.ts` — 8 fixed colors already defined per category (repairs=#EF4444, insurance=#3B82F6, rates=#8B5CF6, utilities=#F59E0B, maintenance=#10B981, cleaning=#06B6D4, management=#6366F1, other=#94A3B8).
- Occupancy chart: filled units = `colors.statusConfirmed`, vacant units = `colors.statusPendingSoft` (light grey fill, so vacant reads as absence).
- Sparkline on property card: `colors.primary` stroke, no fill, 2px stroke width.
- Tooltip bubble background: `colors.surface` with `shadows.sm`. Tooltip border: `colors.border`.
- Reliability score color: on-time % ≥ 90% = `colors.statusConfirmed`; 70–89% = `colors.statusPartial`; < 70% = `colors.statusOverdue`. Applied to the percentage text only.

Source: Locked chart colors from CONTEXT.md D-01, D-02; expense category colors from `lib/expenses.ts`; theme tokens from `constants/theme.ts`.

## Interaction Contracts

### Time Control Bar (D-06, D-07, D-08, D-09)
- Year picker: left/right chevrons. Minimum year = 2000. Maximum year = current year. Active year text: 16px weight 700.
- Granularity toggle: 3 chips — "Yearly" | "Quarterly" | "Monthly". Active chip: `primarySoft` background + `primary` border + `primary` text. Inactive: `surface` background + `border` border + `textSecondary` text.
- When Quarterly selected: Q1 / Q2 / Q3 / Q4 chip row appears below. Active = same pattern as granularity toggle.
- When Monthly selected: 12-month chip row appears (Jan–Dec abbreviations). Active = same pattern. Future months in current year are visually muted (`textDisabled` color) but still tappable (data will be empty).
- All three controls change together before triggering chart re-render. No intermediate loading flash for control state changes — only query re-execution shows skeleton.

### Chart Tap Interaction (D-04)
- Tapping a bar (P&L) or segment (donut) or bar (occupancy): highlights the tapped element (opacity 1.0) and dims all others (opacity 0.4). Shows `ChartTooltip` adjacent to tapped element with exact value.
- Tapping outside or tapping the same element again: deselects, restores all opacity, hides tooltip.
- Tooltip shows: formatted currency (via `formatCurrency()`) for monetary values; percentage for occupancy.
- Tap target minimum: 44px wide × touch area aligned to bar/segment bounds.

### Portfolio → Property Drill-Down (D-11, D-13)
- `PropertyReportCard` is TouchableOpacity with `activeOpacity={0.85}`. Tap navigates to `app/reports/[propertyId].tsx` via `router.push`.
- Back button in DwellaHeader calls `router.back()` — returns to portfolio page with preserved scroll position (no scroll reset).

### Empty State Behavior (D-20, D-21)
- Chart containers render at fixed height even when empty. The axes/frame remain visible. A centered `Text` overlay ("No data for Q3 2026") is rendered inside the chart container at 50% opacity, using `textSecondary` color at 13px (Label scale).
- `EmptyState` component (full icon + text variant) is used only on the portfolio page when the landlord has zero properties.

### Pull-to-Refresh
- Both screens support `RefreshControl` on the outer `ScrollView`. Re-fetches all queries for the current time selection. Spinner color = `colors.primary`.

## Implementation Notes for Executor

1. **No new migrations.** All data is in existing `payments`, `expenses`, `tenants`, `properties` tables. No Edge Functions. All queries are direct Supabase client calls via existing hooks or new aggregation queries protected by RLS.

2. **Composite index already exists.** Migration 019 added a composite index for report aggregate queries. No additional schema work.

3. **INR formatting.** All monetary values use `formatCurrency()` from `lib/utils.ts`. Do not inline currency symbols.

4. **Light/dark.** All new components must be theme-aware via `useTheme()`. No hardcoded hex values. Dark mode is in production.

5. **Realtime not needed.** Reports are aggregate/historical data. No Realtime subscriptions required. Pull-to-refresh is sufficient.

6. **Analytics tracking.** Use `useTrack()` for named events: `report_portfolio_viewed`, `report_property_viewed`, `report_granularity_changed`, `report_chart_tapped`.

7. **victory-native version compatibility.** Install with `npx expo install victory-native` to get Expo-compatible version. Do not use `npm install` directly.

8. **Chart container fixed height.** All chart section cards must use fixed-height containers (`height: 200` for bars, `height: 220` for donut, `height: 180` for occupancy, `height: 40` for sparkline). Do not use `flex: 1` on chart wrappers — height must be predictable for empty-state overlay positioning.

9. **Occupancy chart type decision.** Use stacked bar chart (VictoryStack with VictoryBar) per Claude's discretion. Each period has two bars stacked: filled (bottom, success color) + vacant (top, pendingSoft color). Total bar height = total unit count, always constant. This makes filled vs vacant immediately legible.

10. **EmptyState fontSize override.** The `EmptyState` component defaults to 17px for its title. All uses in this phase must pass `titleStyle={{ fontSize: 16 }}` to stay within the declared 4-size type scale.
