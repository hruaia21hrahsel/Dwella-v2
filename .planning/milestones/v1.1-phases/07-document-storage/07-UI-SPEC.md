## Design System

| Property | Value |
|----------|-------|
| Tool | none — React Native + react-native-paper |
| Preset | not applicable |
| Component library | react-native-paper (Material Design 3) |
| Icon library | MaterialCommunityIcons (@expo/vector-icons) |

**Note:** shadcn is a web-only library and is not applicable to this React Native project. All design tokens are sourced from `constants/theme.ts` (`LightTheme` / `DarkTheme`).

## Spacing Scale

Declared values (multiples of 4 — consistent with existing codebase patterns observed in EmptyState, ConfirmDialog, ListSkeleton):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, tight row spacing |
| sm | 8px | Row item gaps, button row gaps, inline spacing |
| md | 16px | Screen horizontal padding, section internal padding |
| lg | 24px | Card padding, modal padding, section vertical separation |
| xl | 32px | Empty state container padding, major layout gaps |
| 2xl | 48px | n/a for this phase |
| 3xl | 64px | n/a for this phase |

Exceptions:
- Touch targets for icon-only actions (delete, share): minimum 44x44pt (accessibility requirement — wrap icon in a pressable with explicit `hitSlop` of 8pt)
- Full-screen viewer modal: 0px horizontal margin (full bleed), 0px top (status bar handles safe area)
- Document card row height: 72px (sufficient for two-line text + icon, consistent with `TenantCard` row height)
- Skeleton row height: 72px (matches `ListSkeleton` default of 80px adjusted for document card size)

## Typography

All font sizes sourced from observed codebase patterns (`EmptyState`, `ConfirmDialog`, `ProofUploader`) using react-native-paper `Text` variants plus custom `fontSize` where needed. Line heights are explicit React Native values (not ratios).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 20px (1.43) | Document metadata: category badge text, file size, date, subtitle text, error messages |
| Label | 16px | 400 (regular) | 24px (1.5) | Document name in card row, section headers (e.g. "Property Documents"), input field labels |
| Heading | 17px | 700 (bold) | 22px (1.29) | Empty state title, modal title, screen section title — mirrors existing EmptyState title style |
| Display | 20px | 700 (bold) | 28px (1.4) | Screen-level header (DwellaHeader), standalone Documents screen title |

**Weights declared:** 400 (regular) and 700 (bold) only. Two weights maximum — no intermediate weight (600) is used in this phase.

**Note:** react-native-paper variant system (`titleLarge`, `bodyMedium`, `labelMedium`) is used where it matches. Custom `fontSize` + `fontWeight` is used where paper variants do not align (e.g. section headings within a scrollable list).

## Color

All tokens from `constants/theme.ts` via `useTheme()`. Light theme values listed; dark theme equivalents are automatic via the theme system.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `#F5F5F7` (`colors.background`) | `#1f1f1e` | Screen background, full-screen viewer background |
| Secondary (30%) | `#FFFFFF` (`colors.surface`) | `#161616` | Document card background, uploader container, modal surface, name input field background |
| Accent (10%) | `#009688` (`colors.primary`) | `#4DB6AC` | See reserved list below |
| Destructive | `#EF4444` (`colors.error`) | `#F87171` | Delete action button and ConfirmDialog confirm button only |

**Accent reserved for:**
- Upload button ("Upload Document") — contained mode, gradient via `GradientButton`
- Active category filter chip — selected state background
- Document type icon background in card row (using `colors.primarySoft` = `#E0F2F1` as icon wrapper background)
- Progress indicator during upload (ActivityIndicator color)
- Share button in full-screen viewer (text/icon tint)

**Accent is NOT used for:** document card borders, section headers, skeleton shimmer, close button, cancel actions.

**Additional semantic colors used in this phase:**
- `colors.textSecondary` (`#64748B` / `#9E9E9E`) — metadata text (date, file size, category)
- `colors.textDisabled` (`#CBD5E1` / `#555555`) — empty state body text, placeholder text in name input
- `colors.border` (`rgba(0,0,0,0.06)` / `rgba(255,255,255,0.08)`) — card border, input border
- `colors.primarySoft` (`#E0F2F1` / `rgba(77,182,172,0.12)`) — icon wrapper background in document cards and empty state
- `colors.divider` (`#F1F5F9` / `rgba(255,255,255,0.05)`) — section divider between property docs and tenant docs sections

## Interaction Contracts

### Upload Flow
1. User taps "Upload Document" — DocumentUploader modal opens.
2. User taps "Choose File" — expo-document-picker launches (PDF/JPEG/PNG/DOCX).
3. On file selected: file preview appears, name input pre-fills with original filename.
4. User edits name (optional) and selects category.
5. User taps "Upload" — button shows loading state, ActivityIndicator appears.
6. On success: modal closes, document list refreshes, success toast appears ("Document uploaded").
7. On failure: error toast appears ("Upload failed. Please try again."), modal stays open, user can retry.

### View Flow
1. User taps a DocumentCard — DocumentViewer modal opens with loading indicator.
2. Signed URL is generated on tap (not on list load).
3. WebView or Image loads content. On load error: error state with "Could not load document. Tap to retry."
4. User taps Close — modal dismisses.
5. User taps Share Document — expo-sharing opens OS share sheet.

### Delete Flow
1. User long-presses or taps delete icon on DocumentCard (visible only if `uploader_id === current user`).
2. ConfirmDialog appears: "Delete Document? This will permanently remove the file and cannot be undone."
3. User taps "Delete Document" — loading state on confirm button.
4. On success: dialog closes, document removed from list, success toast ("Document deleted").
5. On storage delete failure: dialog closes, error toast ("Could not delete file. Please try again."), document remains in list.

### Category Filter
1. Default filter: "All" (all categories shown).
2. Tap a category chip — list filters to that category only.
3. Active chip: `colors.primary` background, `colors.textOnPrimary` text.
4. Inactive chip: `colors.surface` background, `colors.textSecondary` text, `colors.border` border.
5. Filter is per-section (property docs and tenant docs filtered independently).

## MIME Type Icon Mapping

Document cards display a MIME-type icon using MaterialCommunityIcons. Icon wrapper uses `colors.primarySoft` background, icon tint uses `colors.primary`.

| MIME Type | Icon Name | Label |
|-----------|-----------|-------|
| `application/pdf` | `file-pdf-box` | PDF |
| `image/jpeg` | `file-image` | Image |
| `image/png` | `file-image` | Image |
| `application/msword` | `file-word` | Word |
| `application/vnd.openxmlformats-officedocument.wordprocessingml.document` | `file-word` | Word |
| fallback | `file-document-outline` | File |

Category chip mapping:

| Category | Icon Name |
|----------|-----------|
| lease | `file-sign` |
| id | `card-account-details` |
| insurance | `shield-check` |
| receipts | `receipt` |
