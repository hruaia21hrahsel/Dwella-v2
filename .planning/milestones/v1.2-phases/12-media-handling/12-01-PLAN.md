Create `supabase/functions/whatsapp-media/index.ts` using the standard Deno Edge Function pattern (same imports as other functions: `serve` from `https://deno.land/std@0.177.0/http/server.ts`, `createClient` from `https://esm.sh/@supabase/supabase-js@2`).

**Env vars:** `WHATSAPP_ACCESS_TOKEN`, `ANTHROPIC_API_KEY`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `WHATSAPP_PHONE_NUMBER_ID` (not used directly but available). Derive `WHATSAPP_SEND_URL` as `${SUPABASE_URL}/functions/v1/whatsapp-send`.

**Request interface** (POST body from whatsapp-webhook):
```typescript
interface WhatsAppMediaRequest {
  user_id: string;
  phone: string;
  msg_type: 'image' | 'document';
  media: {
    id: string;
    mime_type?: string;
    caption?: string;
    filename?: string;
    sha256?: string;
  };
}
```

**Implement these functions:**

1. `getMediaDownloadUrl(mediaId: string): Promise<{ url: string; mime_type: string }>` — GET `https://graph.facebook.com/v21.0/${mediaId}` with `Authorization: Bearer ${WHATSAPP_ACCESS_TOKEN}`. Returns `{ url, mime_type }`.

2. `downloadMediaBinary(url: string): Promise<ArrayBuffer>` — GET the CDN URL with same bearer token. Returns ArrayBuffer. Both steps must happen immediately in sequence (CDN URL expires in ~5 minutes).

3. `getExtFromMimeType(mime: string, filename?: string): string` — Map MIME to extension:
   - `image/jpeg` -> `jpg`, `image/png` -> `png`, `image/webp` -> `webp`
   - `application/pdf` -> `pdf`, `application/msword` -> `doc`
   - `application/vnd.openxmlformats-officedocument.wordprocessingml.document` -> `docx`
   - Fallback: if `application/octet-stream` and filename ends with `.pdf`, return `pdf` (Pitfall 4)
   - Final fallback: extract from filename extension, or `bin`

4. `findTenantForUser(supabase, userId: string)` — Query `tenants` table where `user_id = userId` and `is_archived = false`, selecting `id, property_id, tenant_name, properties(owner_id)`. If exactly 1 result, return it. If multiple, check current month's `payments` for `status IN ('pending', 'partial')` — return the tenant with an open payment. Fallback: return first tenant.

5. `sendWhatsApp(to: string, text: string)` — POST to `WHATSAPP_SEND_URL` with `{ to, type: 'text', text }` and `Authorization: Bearer ${SUPABASE_SERVICE_KEY}`. Same pattern as whatsapp-webhook line 18-31.

6. `classifyPaymentProof(imageBase64: string, mimeType: string, caption: string, userContext: string)` — POST to `https://api.anthropic.com/v1/messages` with:
   - `model: 'claude-sonnet-4-20250514'`, `max_tokens: 256`
   - `messages` array with one user message containing TWO content blocks:
     - `{ type: 'image', source: { type: 'base64', media_type: mimeType, data: imageBase64 } }`
     - `{ type: 'text', text: 'User context...\n\nUser caption: "${caption}"\n\nIs this a rent payment proof screenshot/photo? If yes, identify: tenant_name (match to context), month (1-12), year (4 digits), amount (number if visible). Reply ONLY with JSON: {"is_payment_proof": bool, "tenant_name": string|null, "month": number|null, "year": number|null, "amount": number|null}' }`
   - Headers: `Content-Type: application/json`, `x-api-key: ${ANTHROPIC_API_KEY}`, `anthropic-version: 2023-06-01`
   - Parse response: extract JSON from `data.content[0].text` using regex `/{[\s\S]*}/`
   - Return `{ is_payment_proof: boolean; tenant_name?: string; month?: number; year?: number; amount?: number }`

7. **Copy `buildContext()` from process-bot-message/index.ts** (lines 368-431) including `sanitizeForContext()` helper (lines 357-363). This provides tenant names and payment status as context for Claude vision classification.

8. **Main handler (`serve`):**
   - CORS preflight (same pattern as other functions)
   - Parse JSON body as `WhatsAppMediaRequest`
   - Validate: `user_id`, `phone`, `msg_type`, `media.id` must be present
   - Create Supabase service-role client
   - Call `getMediaDownloadUrl(media.id)` then `downloadMediaBinary(url)` immediately in sequence
   - Resolve effective MIME: prefer response's `mime_type`, fall back to `media.mime_type`, apply octet-stream filename fallback
   - **If `msg_type === 'image'`:**
     - Convert ArrayBuffer to base64: `btoa(String.fromCharCode(...new Uint8Array(buffer)))`. Note: for large buffers, chunk the conversion to avoid stack overflow — process in 8192-byte slices.
     - Call `buildContext(supabase, user_id)` for Claude vision context
     - Call `classifyPaymentProof(base64, mimeType, media.caption ?? '', context)`
     - If `is_payment_proof === true`:
       - Call `findTenantForUser(supabase, user_id)`
       - If no tenant found: reply "I couldn't find a linked tenant account for your number. Link your account first in the Dwella app."
       - Determine month/year: prefer classification result, fall back to current month/year
       - Storage path: `${tenant.property_id}/${tenant.id}/${year}-${String(month).padStart(2,'0')}.jpg`
       - Upload to `payment-proofs` bucket with `{ contentType: mimeType, upsert: true }`
       - Update `payments` table: `.update({ proof_url: storagePath, notes: 'Proof attached via WhatsApp' }).eq('tenant_id', tenant.id).eq('month', month).eq('year', year)`. If no payment row exists, create one with `status: 'paid'`, `amount_paid: classification.amount ?? tenant.monthly_rent ?? 0`, `amount_due: tenant.monthly_rent ?? 0`, `due_date` computed from tenant.due_day
       - Build month name using `new Date(year, month-1).toLocaleString('en', { month: 'long' })`
       - Reply: `Got it! I've attached your payment photo as proof for ${monthName} ${year}. Your payment is now recorded as "${newStatus}".`
     - If `is_payment_proof === false`:
       - Reply: `I couldn't tell if this photo is a payment receipt. If it is, add a caption like "payment for March" and send again, or log the payment via text: "Mark Ravi's March rent as paid"`
   - **If `msg_type === 'document'`:**
     - Call `findTenantForUser(supabase, user_id)` — for documents, also check if user is a property owner (query `properties` where `owner_id = user_id` and `is_archived = false`). If tenant found, use tenant's property_id and tenant_id. If no tenant but is owner, use first owned property_id and `tenant_id: null` (property-level doc).
     - If neither tenant nor owner: reply "I couldn't find a linked tenant account for your number. Link your account first in the Dwella app."
     - Generate UUID filename: `${crypto.randomUUID()}.${getExtFromMimeType(effectiveMime, media.filename)}`
     - Storage path: `${propertyId}/${tenantId}/${filename}` (or `${propertyId}/property/${filename}` if no tenant)
     - Upload to `documents` bucket with `{ contentType: effectiveMime }`
     - Insert `documents` row: `{ property_id: propertyId, tenant_id: tenantId ?? null, uploader_id: user_id, name: media.filename ?? filename, category: 'other', storage_path: storagePath, mime_type: effectiveMime, file_size: buffer.byteLength }`
     - Reply: `Your document "${media.filename ?? filename}" has been saved to your property files. You can view it in the Dwella app under Documents.`
   - Wrap entire handler in try/catch. On error: log with `console.error`, reply: "Something went wrong processing your media. Please try again or use the Dwella app directly." Always return HTTP 200 with `{ success: true }` or `{ error: string }`.
