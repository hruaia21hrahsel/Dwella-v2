**hooks/useReportData.ts:**

Export two hooks:

**`usePortfolioData(year: number)`:**
- Fetch owned properties with tenants: `supabase.from('properties').select('id, name, total_units, tenants(id, tenant_name, flat_no, lease_start, lease_end, is_archived, due_day)').eq('owner_id', user.id).eq('is_archived', false)`
- Fetch all payments for the selected year: collect all tenant IDs from all properties, then `supabase.from('payments').select('*').in('tenant_id', allTenantIds).eq('year', year)`. If allTenantIds is empty, skip query and use empty array.
- Fetch all expenses: `supabase.from('expenses').select('*').eq('user_id', user.id)` (all expenses, filtered client-side by period)
- Use `useMemo` to run `aggregatePortfolio(properties, allPayments, allExpenses, { year, granularity: 'yearly' })` — portfolio always uses yearly granularity for KPIs
- Return: `{ kpis: PortfolioKPIs; summaries: PropertySummary[]; properties; isLoading; error; refresh }`
- No Realtime subscriptions (per UI-SPEC note 5). Pull-to-refresh only.
- Memoize aggregation results with `useMemo` keyed on `[properties, payments, expenses, year]`.

**`usePropertyReportData(propertyId: string, period: TimePeriod)`:**
- Fetch single property with tenants: `supabase.from('properties').select('id, name, total_units, tenants(id, tenant_name, flat_no, lease_start, lease_end, is_archived, due_day, monthly_rent)').eq('id', propertyId).single()`
- Fetch payments for property's tenants for the selected year: tenant IDs from property, then `supabase.from('payments').select('*').in('tenant_id', tenantIds).eq('year', period.year)`
- Fetch expenses for property: `supabase.from('expenses').select('*').eq('property_id', propertyId)`
- Use `useMemo` to run all 4 aggregation functions:
  - `aggregateByPeriod(payments, expenses, period)` → P&L data
  - `aggregateByCategory(expenses, period)` → donut data
  - `calcReliability(payments, tenants, period)` → reliability data
  - `calcOccupancy(tenants, totalUnits, period)` → occupancy data
- Compute `hasData` flags: `plHasData = payments.length > 0 || expenses.length > 0` (for the full year, not just selected period). Per-chart hasData should check if the specific period has any rows.
- Return: `{ propertyName, plData, categoryData, reliabilityData, occupancyData, totalExpenses, plHasData, categoryHasData, reliabilityHasData, occupancyHasData, isLoading, error, refresh }`
- Memoize all aggregation with `useMemo` keyed on `[property, payments, expenses, period]`.
- Re-fetch raw data only when `propertyId` changes or on pull-to-refresh. Period changes only re-compute aggregations client-side (no new queries).
