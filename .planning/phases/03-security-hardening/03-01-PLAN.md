Per D-04, D-05, D-06:

1. Add a UUID v4 regex constant at the top of the file (after the store URL constants, before `Deno.serve`):
```typescript
const UUID_V4_REGEX = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;
```

2. Immediately AFTER the existing `if (!token)` check (line 27-29), add a UUID validation check:
```typescript
if (!UUID_V4_REGEX.test(token)) {
  return new Response(
    '<!DOCTYPE html><html lang="en"><head><meta charset="UTF-8"/><meta name="viewport" content="width=device-width,initial-scale=1.0"/><title>Invalid Invite</title></head><body style="font-family:-apple-system,sans-serif;display:flex;justify-content:center;align-items:center;min-height:100vh;background:#F8FAFC"><div style="text-align:center;padding:2rem"><h1 style="color:#EF4444">Invalid Invite Link</h1><p>This invite link is malformed. Please ask your landlord for a new invite.</p></div></body></html>',
    { status: 400, headers: { 'Content-Type': 'text/html; charset=utf-8' } },
  );
}
```

3. This placement is BEFORE the `isAndroid` check (line 32), which means BOTH the Android intent redirect path AND the iOS HTML template path are protected. The token now only contains hex characters and hyphens when it reaches any interpolation point (lines 38, 46, 187, 193).

4. Do NOT add any additional HTML encoding/escaping — per D-06, the UUID regex is sufficient because hex+hyphens are HTML-safe by construction.
