**Step 2: Create `app/documents/index.tsx` — Standalone Documents Screen**

This is the main documents screen accessed from the tools menu. Per UI-SPEC Screen 1:

State management:
- `selectedPropertyId` (string | null) — defaults to first owned property
- `selectedCategory` (DocumentCategory | null) — defaults to null (All)
- `uploaderVisible` (boolean)
- `viewerDoc` (Document | null) — document to view, null closes viewer
- `deleteDoc` (Document | null) — document to confirm delete
- `uploadTenantId` (string | null) — which tenant scope to upload to (null = property-wide)

Hooks:
- `useProperties()` for property picker options
- `useDocuments(selectedPropertyId)` for all documents in selected property (no tenantId filter — gets all)
- `useTenants(selectedPropertyId)` for tenant list (to render tenant sections for landlord)
- `useAuthStore()` for current user

Determine role: check if current user is owner of selected property (`property.owner_id === user.id`).

Layout per UI-SPEC:
```
DwellaHeader (title: "Documents")
Property picker (if user has multiple properties):
  - Use a Picker or modal selector showing property names
  - Default to first property
  - On change: update selectedPropertyId
─────────────────────────────────────
CategoryFilterBar (selected, onSelect → setSelectedCategory)
─────────────────────────────────────
ScrollView:
  Section: "Property Documents"
    Filter documents where tenant_id is null AND (selectedCategory === null OR category === selectedCategory)
    Render DocumentCard for each, or EmptyState:
      - Landlord empty: icon "file-document-outline", title "No property documents yet", message "Upload leases, insurance, or other files for this property.", actionLabel "Upload Document", onAction opens uploader with tenantId=null
      - Tenant empty: icon "file-document-outline", title "No property documents yet", message "Your landlord hasn't uploaded any property documents yet." (no action button)
  ─────────────────────────────────────
  Divider (colors.divider, height 1)
  ─────────────────────────────────────
  If landlord:
    Section: "Tenant Documents"
    For each tenant from useTenants:
      Sub-header: tenant.tenant_name (16px, bold)
      Filter documents where tenant_id === tenant.id AND category filter
      Render DocumentCard for each, or EmptyState per tenant:
        title "No documents uploaded", message "No documents have been uploaded for this tenant."
  If tenant:
    Section: "My Documents"
    Find current user's tenant record in the property
    Filter documents where tenant_id matches
    Render DocumentCard for each, or EmptyState:
      title "No documents uploaded", message "Upload your ID, lease, or other documents for your tenancy.", actionLabel "Upload Document"
─────────────────────────────────────
```

FAB or bottom button:
- Landlord: "Upload Document" — opens uploader. If in tenant section, set `uploadTenantId` to that tenant's ID; if in property section, set to null.
- Tenant: "Upload to My Tenancy" — opens uploader with tenantId = user's tenant record ID.

ConfirmDialog for delete:
- visible when `deleteDoc !== null`
- title "Delete Document?"
- message "This will permanently remove the file and cannot be undone."
- confirmLabel "Delete Document"
- destructive: true
- onConfirm: call `deleteDocument(deleteDoc)`, then toast "Document deleted" on success or "Could not delete file. Please try again." on error. Set `deleteDoc = null`.
- onCancel: `setDeleteDoc(null)`

Loading state: Show `ListSkeleton` with `count={3}` and `rowHeight={72}` while `isLoading`.

DocumentUploader: `visible={uploaderVisible}`, `propertyId={selectedPropertyId}`, `tenantId={uploadTenantId}`, `onUploadComplete={() => refresh()}`, `onClose={() => setUploaderVisible(false)}`.

DocumentViewer: `visible={viewerDoc !== null}`, `document={viewerDoc}`, `onClose={() => setViewerDoc(null)}`.
