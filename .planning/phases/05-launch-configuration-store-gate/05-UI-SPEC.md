## Component Inventory

New components this phase produces:

### `components/AiDisclosureModal.tsx`

**Pattern**: Mirrors `components/ConfirmDialog.tsx` exactly — `Portal > Modal > Surface` structure.

**Key constraints:**
- `dismissable={false}` — user MUST tap "I Understand"; backdrop tap must not dismiss
- Renders `null` when `aiDisclosureAccepted === true` — no DOM overhead for returning users
- Placed at the top of each AI entry screen render tree (not in `_layout.tsx`)
- Entry screens: `app/(tabs)/bot/index.tsx`, `app/tools/ai-insights.tsx`, `app/tools/ai-search.tsx`, `app/tools/smart-reminders.tsx`, `components/AiInsightCard.tsx`

**Structure:**
```
Portal
  Modal (visible={!aiDisclosureAccepted}, dismissable={false}, contentContainerStyle={{ margin: 24 }})
    Surface (borderRadius: 16, padding: 24, gap: 16, backgroundColor: colors.surface)
      Text variant="titleLarge" (weight 700, color: colors.textPrimary)
      Text variant="bodyMedium" (color: colors.textSecondary, lineHeight: 22)
      Button mode="contained" (buttonColor: colors.primary)
```

Source: CONTEXT.md locked decision + RESEARCH.md Pattern 2 + `ConfirmDialog.tsx` established pattern

---

### `components/UpdateGate.tsx`

**Pattern**: Wrapper component rendered in `app/_layout.tsx` that passes `{children}` through unless a store update is needed.

**Key constraints:**
- No-op guard: `if (!Updates.isEnabled) return <>{children}</>` — must not crash in Expo Go or development
- Shows blocking screen only when `isUpdateAvailable === true` AND `Updates.fetchUpdateAsync()` fails (fingerprint mismatch)
- Auto-applies compatible OTA updates silently: fetch on `isUpdateAvailable`, reload on `isUpdatePending`
- Store redirect uses `expo-linking` `Linking.openURL()` to App Store / Play Store

**Structure (blocking screen only, shown on fingerprint-incompatible update):**
```
View (flex: 1, backgroundColor: colors.background, justifyContent: center, padding: 24)
  Surface (borderRadius: 16, padding: 24, gap: 16, backgroundColor: colors.surface)
    Text variant="titleLarge" (weight 700, color: colors.textPrimary)
    Text variant="bodyMedium" (color: colors.textSecondary, lineHeight: 22)
    Button mode="contained" (buttonColor: colors.primary) — opens store URL
```

Source: CONTEXT.md locked decision + RESEARCH.md Pattern 4

## Interaction Contracts

### AiDisclosureModal — First-Use Gate

| Trigger | Show when `aiDisclosureAccepted === false` and user navigates to any AI screen |
|---|---|
| Dismissal | Only via "I Understand" button — `dismissable={false}` |
| State transition | On button press: `setAiDisclosureAccepted(true)` → modal unmounts (`return null`) |
| Persistence | Zustand `partialize` via AsyncStorage — survives cold restart; user sees modal exactly once |
| Non-AI screens | Modal never renders — not in `_layout.tsx` |
| Re-show condition | Never (state is persisted; only way to re-trigger is clearing AsyncStorage) |

### UpdateGate — OTA Update Lifecycle

| State | Behavior |
|---|---|
| `!Updates.isEnabled` (dev / Expo Go) | No-op: return `{children}` immediately |
| `isUpdateAvailable === true` | Call `Updates.fetchUpdateAsync()` silently |
| `isUpdatePending === true` | Call `Updates.reloadAsync()` — applies update transparently |
| `fetchUpdateAsync()` throws (fingerprint mismatch) | Set local error state → render forced-update blocking screen |
| Forced-update screen CTA tapped | `Linking.openURL()` with platform-specific store URL |

### Privacy Checklist (LAUNCH-01)

No interactive UI — this is a markdown document (`docs/privacy-checklist.md`) used for manual App Store Connect entry. No design contract needed beyond document structure.

### EAS Config Validation (LAUNCH-03)

No UI — terminal-only operation. No design contract.
