  <action>
(EAS-08) Run the EAS build command to trigger a cloud build and auto-submit to App Store Connect:

```bash
eas build --platform ios --profile production --auto-submit --non-interactive
```

If `--non-interactive` fails because credentials need interactive setup, fall back to:
```bash
eas build --platform ios --profile production --auto-submit
```

If the build command prompts about Apple credentials (distribution certificate, provisioning profile), follow the interactive prompts — EAS manages these automatically for managed workflow apps.

If the build command fails with an `ITMS-91053` error (missing privacy manifest entries), note the specific missing API types in the summary. The privacy manifest in app.json may need additional entries beyond `NSPrivacyAccessedAPICategoryUserDefaults`.

Monitor the build URL printed by EAS CLI. The cloud build typically takes 10-15 minutes. The auto-submit step uploads the .ipa to App Store Connect, which then takes another 5-15 minutes for TestFlight processing.

If the submit step fails because no ASC API key is stored, the CLI will prompt to register one. The .p8 key file is at: `C:/Users/Spongeass/Downloads/AuthKey_5ZP7RY3DNV.p8` (Issuer ID and Key ID will be prompted — Key ID is `5ZP7RY3DNV`, Team ID is `35G5HXD9K9`).
