**Imports and env vars** (match existing project patterns):
```typescript
import { serve } from 'https://deno.land/std@0.177.0/http/server.ts';

const WHATSAPP_ACCESS_TOKEN = Deno.env.get('WHATSAPP_ACCESS_TOKEN')!;
const WHATSAPP_PHONE_NUMBER_ID = Deno.env.get('WHATSAPP_PHONE_NUMBER_ID')!;
```

**Request body schema** — discriminated union on `type` field (per D-07):
```typescript
interface WhatsAppSendRequest {
  to: string;            // E.164 phone number
  type: 'text' | 'template' | 'interactive' | 'document';
  text?: string;
  template?: {
    name: string;
    language?: string;   // Default 'en'
    components?: Array<{
      type: 'body' | 'header' | 'button';
      parameters: Array<{ type: 'text'; text: string }>;
    }>;
  };
  interactive?: {
    body: string;
    buttons: Array<{ id: string; title: string }>;
  };
  document?: {
    link: string;
    filename?: string;
    caption?: string;
  };
}
```

**Meta API payload construction** — build the correct payload shape per type:
- `text`: `{ messaging_product: 'whatsapp', to, type: 'text', text: { body: req.text } }`
- `template`: `{ messaging_product: 'whatsapp', to, type: 'template', template: { name: req.template.name, language: { code: req.template.language ?? 'en' }, components: req.template.components ?? [] } }`
- `interactive`: `{ messaging_product: 'whatsapp', to, type: 'interactive', interactive: { type: 'button', body: { text: req.interactive.body }, action: { buttons: req.interactive.buttons.map(b => ({ type: 'reply', reply: { id: b.id, title: b.title } })) } } }`
- `document`: `{ messaging_product: 'whatsapp', to, type: 'document', document: { link: req.document.link, filename: req.document.filename, caption: req.document.caption } }`

**Retry logic** (per D-08): Retry once on HTTP 429 or 5xx with 1-second delay. Fail after second attempt.
```typescript
async function callMetaAPI(payload: object): Promise<Response> {
  const doFetch = () =>
    fetch(`https://graph.facebook.com/v21.0/${WHATSAPP_PHONE_NUMBER_ID}/messages`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        Authorization: `Bearer ${WHATSAPP_ACCESS_TOKEN}`,
      },
      body: JSON.stringify(payload),
    });

  let res = await doFetch();
  if (res.status === 429 || res.status >= 500) {
    await new Promise(r => setTimeout(r, 1000));
    res = await doFetch();
  }
  return res;
}
```

**CORS handling**: Return CORS preflight response for OPTIONS requests (same pattern as whatsapp-send-code):
```typescript
headers: {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Headers': 'authorization, x-client-info, apikey, content-type',
}
```

**Response format**: Always return HTTP 200 with JSON body. On success: `{ success: true }`. On failure: `{ error: "description", status: metaStatusCode }`. This is so callers (other Edge Functions) don't need to handle unexpected HTTP errors.

**Validation**: Return `{ error: "..." }` (HTTP 200) if:
- `to` is missing or empty
- `type` is not one of the 4 valid types
- Required sub-field is missing for the given type (e.g., `text` type without `text` field)

**Logging**: Log errors with `console.error('whatsapp-send error:', ...)` including the Meta API response body on failure. Log successful sends with `console.log('whatsapp-send: sent ${type} to ${to}')`.

**Document structure** (exactly these sections, in order):

**# Meta WhatsApp Business API Setup Guide**

**## Prerequisites**
- Meta Business Account (business.facebook.com)
- A phone number NOT currently registered on WhatsApp (will be registered as business number)
- Supabase project with Edge Functions enabled

**## 1. Create Meta App**
- Go to developers.facebook.com -> My Apps -> Create App
- Select "Business" type
- Enter app name (e.g., "Dwella WhatsApp Bot")
- Select your Business Account

**## 2. Add WhatsApp Product**
- In App Dashboard -> Add Products -> WhatsApp -> Set Up
- This creates a WhatsApp Business Account linked to your app

**## 3. Register Phone Number**
- WhatsApp -> Getting Started -> Add Phone Number
- Complete phone verification (SMS or voice call)
- Or use the test number provided by Meta for development

**## 4. Get Phone Number ID**
- WhatsApp -> Getting Started page shows your Phone Number ID
- Save this value — it becomes `WHATSAPP_PHONE_NUMBER_ID`

**## 5. Create System User** (per D-03 — permanent token, NOT developer token)
- Business Settings -> Users -> System Users -> Add
- Name: "Dwella Bot" (or any descriptive name)
- Role: Admin
- IMPORTANT: A temporary developer token expires every 24 hours. The System User token is permanent and required for production.

**## 6. Assign App to System User**
- In the System User page -> Assign Assets -> Apps tab
- Select your Dwella app -> Full Control -> Save

**## 7. Assign WhatsApp Account to System User**
- Business Settings -> Accounts -> WhatsApp Accounts
- Select your account -> People tab -> Add People
- Add the System User created in step 5 with Full Control
- CRITICAL: Without this step, the token will generate but API calls return 403

**## 8. Generate Permanent Token**
- System Users page -> select your System User -> Generate Token
- Select your app
- Check permissions: `whatsapp_business_messaging` AND `whatsapp_business_management`
- Generate -> copy the token
- Save as `WHATSAPP_ACCESS_TOKEN`

**## 9. Get App Secret**
- App Dashboard -> Settings -> Basic -> App Secret (click "Show")
- Save as `WHATSAPP_APP_SECRET`

**## 10. Set Verify Token**
- Choose any random string (e.g., generate with `openssl rand -hex 32`)
- Save as `WHATSAPP_VERIFY_TOKEN`

**## 11. Configure Webhook**
- WhatsApp -> Configuration -> Edit Webhook
- Callback URL: `https://<YOUR_SUPABASE_URL>/functions/v1/whatsapp-webhook`
- Verify Token: the value from step 10
- Subscribe to webhook field: `messages`

**## 12. Environment Variables**

Table with columns: Variable | Where to Get | Where to Set

| `WHATSAPP_ACCESS_TOKEN` | Step 8: System User token | Supabase Dashboard -> Edge Functions -> Secrets |
| `WHATSAPP_PHONE_NUMBER_ID` | Step 4: WhatsApp Getting Started | Supabase Dashboard -> Edge Functions -> Secrets |
| `WHATSAPP_VERIFY_TOKEN` | Step 10: Your chosen string | Supabase Dashboard -> Edge Functions -> Secrets |
| `WHATSAPP_APP_SECRET` | Step 9: App Settings -> Basic | Supabase Dashboard -> Edge Functions -> Secrets |
| `EXPO_PUBLIC_WHATSAPP_BOT_PHONE` | Your registered number in E.164 format (e.g., +919999999999) | `.env` file in project root |

**## 13. Deploy Edge Functions**
```bash
supabase functions deploy whatsapp-webhook
supabase functions deploy whatsapp-send
```
