  <action>
1. Create metro.config.js at the repo root (per D-04). This file does NOT exist yet — create it fresh:
   ```javascript
   const { getDefaultConfig } = require('expo/metro-config');

   /** @type {import('expo/metro-config').MetroConfig} */
   const config = getDefaultConfig(__dirname);

   // Exclude the /website Next.js project from Metro's module resolution.
   // Without this, Metro may attempt to resolve Next.js modules and fail.
   config.resolver.blockList = [
     ...(config.resolver.blockList ?? []),
     /\/website\/.*/,
   ];

   module.exports = config;
   ```
   IMPORTANT: Always spread the existing blockList first. Expo populates it with internal exclusions. Replacing it entirely would break Expo.

2. Edit tsconfig.json at the repo root (per D-05). Add "website" to the existing exclude array. The full exclude array should become:
   ```json
   "exclude": [
     "supabase/functions",
     "website"
   ]
   ```
   Do NOT change any other field in tsconfig.json. The include, compilerOptions, and extends fields remain exactly as they are.

3. Edit package.json at the repo root (per D-06). Add two convenience scripts to the existing scripts section:
   ```json
   "website:dev": "npm --prefix website run dev",
   "website:build": "npm --prefix website run build"
   ```
   Add these AFTER the existing "test" script entry. Do NOT modify any other scripts or fields.

4. Verify root TypeScript checking ignores website:
   ```bash
   npx tsc --noEmit
   ```
   This must pass without any errors referencing Next.js modules (no "Cannot find module 'next'" errors).
