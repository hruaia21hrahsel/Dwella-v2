Create website/vercel.json with framework and build settings as code (per D-08):

```json
{
  "framework": "nextjs",
  "buildCommand": "npm run build",
  "outputDirectory": "out",
  "installCommand": "npm install"
}
```

Key details:
- `"framework": "nextjs"` tells Vercel to use the Next.js build adapter
- `"buildCommand": "npm run build"` runs `next build` which produces static output due to `output: 'export'` in next.config.ts
- `"outputDirectory": "out"` points to the static export output (NOT `.next/` — that is the server build output)
- `"installCommand": "npm install"` ensures dependencies are installed in the website/ context

This is a human-action checkpoint. The user must configure Vercel via the dashboard — these steps cannot be automated via CLI because the Vercel project does not exist yet.

Steps for the user:
1. Go to https://vercel.com/new and import the `hruaia21hrahsel/Dwella-v2` GitHub repository
2. Set Root Directory to `website`
3. Vercel auto-detects Next.js framework from vercel.json — click Deploy
4. After deploy succeeds, go to Project Settings > Git > Ignored Build Step
5. Enter exactly: `git diff HEAD^ HEAD --quiet -- .`
6. Verify deployment is live at the assigned URL (e.g., dwella-xxx.vercel.app)

The Ignored Build Step command semantics (per D-07):
- `git diff HEAD^ HEAD --quiet -- .` runs from website/ context (Root Directory)
- Exit 0 (no diff in website/) = skip build
- Exit 1 (website/ files changed) = proceed with build
  </action>
